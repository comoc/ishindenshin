# 意思伝達Webアプリケーション 基本仕様書（スイッチ入力特化版）

## 1. アプリケーション概要
筋萎縮性側索硬化症（ALS）や多系統萎縮症（MSA）などの進行性の神経難病のある方、または四肢麻痺などで身体の動作が極めて限定的な方のためのWebアプリケーションです [1]。**たった一つのスイッチ入力（走査入力）のみ**で、意思伝達やメール、家電操作などの機能をコントロールできるように設計します [1]。

## 2. 対象ユーザー
* **利用者（当事者）**: わずかな身体動作のみが可能で、接点式などの外部スイッチを1つ操作できる方 [1]。
* **介護者・支援者**: 利用者の初期設定や、状態に合わせた走査スピード等の微調整をサポートする方 [1, 2]。

## 3. 入力インターフェース要件：スイッチ入力（走査入力）
視線入力機能はオフにし、スイッチによる走査入力のみで操作を完結させます [3]。Webブラウザ上では、外部スイッチからの入力信号を特定のキーボード入力（SpaceキーやEnterキーなど）として受け取る前提とします。

### 3.1 基本的な操作フロー
1. 画面上で、**えんじ色の枠が自動的に縦方向に移動（走査）**を始めます [3]。
2. 入力したい文字や機能の位置に枠が来たタイミングで、利用者がスイッチを押します [3]。
3. 続いて、**枠が横方向の移動（走査）**に切り替わります [3]。
4. 目的の文字や機能の位置に枠が来たタイミングで再度スイッチを押すことで、選択した文字が入力されます [3]。

### 3.2 入力効率化機能
少ない操作回数で快適に入力できるよう、以下の補助機能を実装します。
* **濁音・半濁音の変換機能**: 濁音や半濁音などの文字を持つひらがなを選択した場合、**続けてスイッチを押すことで文字の種類を切り替える**ことができます [4]。
* **走査方向の反転機能**: タイミングが合わず目的の文字を通り過ぎてしまった場合、**走査枠が画面の端に来た際にスイッチを押すことで走査方向が反転**します [4]。これにより、素早く入力を続けることが可能です [4]。

## 4. 主要機能要件
すべての機能は、上記の「縦・横の走査＋スイッチ決定」でアクセス可能なUIパネルとして配置します。

### 4.1 コミュニケーション機能
* **文章の作成と保存**: スイッチ操作で文字盤から文章を作成し、保存します [5]。
* **音声合成**: 作成した文章を合成音声で発声（読み上げ）します [5]。
* **絵文字入力**: 130種類以上の絵文字を搭載し、複数の絵文字を組み合わせることで少ない入力でニュアンスや意思を伝えられるようにします [2]。

### 4.2 ネットワーク・外部連携機能
* **メール・SNS機能**: Webアプリの画面内から、電子メールの送受信やSNSによるコミュニケーションを走査入力で実行できるようにします [2, 5]。

### 4.3 外部機器連携（環境制御）
* **家電操作**: 画面上のパネルからテレビやビデオ、エアコン、照明などの家電を操作し、ご自身でできることを増やします [2]。
  * *(※Webアプリとしての実装補足: オリジナルのように付属の赤外線リモコンを経由するのではなく、スマートリモコンのWeb APIを呼び出す形で代替実装します)*
* **印刷機能**: 作成したテキストをプリンターで印刷できるようにします [1]。

## 5. UI/UX・デザイン要件
### 5.1 利用者向けUI（走査向けデザイン）
* **パネルとアイコンによる統一デザイン**: 直感的に使いやすく、大きく見やすい文字とパネル・アイコンによる統一性のあるデザインを採用します [1]。
* **安全な終了機能**: 画面上の「電源アイコン」を選択することで終了の確認画面へ移行し、安全にアプリを終了（電源オフ）できるようにします [4]。
